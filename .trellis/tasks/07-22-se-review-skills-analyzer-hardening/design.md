# Harden The Skill Review Inventory Analyzer Design

## Overview

The deterministic analyzer already bounds discovery and records provenance, but
four contract edges remain: `zip(strict=True)` exceeds the documented Python
3.9 floor; top-level string hits are called pinned tests without semantic
proof; unresolved installed copies can still report `changeable=true`; and
related-template recursion can hash generated bytecode.

## Proposal

- Keep Python 3.9 as the minimum and remove the Python 3.10-only `zip` keyword.
- Bump the public inventory schema to version 3 and rename `pinnedTests` to
  `testTextReferences`. State that these locators are substring candidates,
  not proof of behavioral assertions.
- Compute `changeable` only when owner identity is verified, an allowed
  canonical template root exists, and the canonical source is inside it.
  Preserve `reviewable=true` for safely read unowned copies.
- Apply one ignored-path predicate to related-template hashing, excluding
  ignored directories, `__pycache__`, and `*.pyc` files.

## Boundaries

- Do not execute reviewed content or widen discovery roots.
- Do not make unresolved or generic copies task-routable.
- Do not infer behavioral coverage from names or string matches.
- Update the canonical skill, analyzer, schema consumers, tests, and release
  surfaces together.

## Risks And Validation

Schema renaming can break consumers, ownership tightening can disable valid
mutation, and ignore filters can omit authored files. Focused tests will pin
schema 3, Python 3.9-compatible constructs, conservative test references,
reviewable-but-non-changeable unowned copies, and snapshot stability under
bytecode-only changes. Existing inventory, drift, symlink, and routing tests
must remain green before the full repository and PR gates.
