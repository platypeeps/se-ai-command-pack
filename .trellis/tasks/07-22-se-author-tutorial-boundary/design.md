# Author And Tutorial Routing Boundary Design

## Overview

Both skills intentionally accept tutorials, but they own different outcomes.
`se-author` develops an original publication contribution, while `se-tutorial`
develops executable teaching from a known state to a verified result. The
current trigger text does not make that shared word safe to route.

## Proposal

- Route article-shaped tutorials centered on an original thesis, argument,
  experience, or publication contribution to `se-author`.
- Route ordered technical teaching centered on completing and verifying an
  observable result to `se-tutorial`.
- When both remain plausible, ask for the intended reader outcome rather than
  removing tutorial capability from either skill or selecting arbitrarily.
- Preserve each skill's existing research, editorial, safety, checkpoint,
  execution-state, recovery, cleanup, and publication boundaries.

## Validation

Focused contract pins will cover both positive routes and the ambiguous prompt
path. The normal generator, release, and full repository gates must remain
green.
